Note the @input.name here, is declaring the name space of the input which will be composed of a <name, value> pair and passed to the Client component as parameters.

Data binding

Now we can input the hero name and description, but may wanna make things more smart as: "Autocomplete a description by enter the name".

How to achieve this? Use the data binding things gonna to be easy.

Let's bind @input.text under hero-new.form.plist > Item 1 > properties:

KeyTypeValue
@input.textString%!(Description of %@),=>name

The expression details:

  1. Use a =>name expression, = refer to oneway binding and >name refer to the text of the name input
  2. Use a %! tag, this ensures the description will not become to be Description of but as empty while the >name is empty.

Submitting

Let's add a submit button for the form by creating a submit component submit.input.plist:

KeyTypeValue
RootDictionary(3 items)
viewsDictionary(1 item)
buttonDictionary(2 items)
clazzStringPBButton
propertiesDictionary(2 items)
clipsToBoundsBooleanYES
layer.cornerRadiusNumber5
metricsDictionary(2 items)
leftString12
topString24
formatsArray(2 items)
Item 0String|-left-[button]-left-|
Item 1StringV:|-top-[button]-top-|

We used PBButton here which base to UIButton but support the automatical color change of highlighted, disabled state and also support configure the properties of title, action and so on, you can see the details in PBButton component.

Modify the hero-new.form.plist and append a new item for rows:

KeyTypeValue
Item 2Dictionary(2 items)
layoutStringsubmit.input
propertiesDictionary(2 items)
@button.backgroundColorString#2baf2b
@button.titleStringSubmit

Now we needs to accomplish a series of interactions by click the submit button:

  1. Validate the form
  2. Send a POST request
  3. Back to the heroes list page
  4. Append a new row to display the created hero

The related action can be described as submit -> post -> pop -> addRowã€‚

Let's configure the action for the submit button properties:

KeyTypeValue
@button.actionDictionary(2 items)
typeStringsubmit
next.doneDictionary(3 items)
typeStringpost
nameStringheroes
next.doneDictionary(2 items)
typeStringpop
next.doneDictionary(1 item)
typeStringaddRow

As we see, the pipeline of action doesn't contain a redundant data configuration, owe to the design of PBAction, we pipe the stream automatically by manage the action state:

  1. submit Validate the form, collect the <name, value> pairs and save to state.params
  2. post send the request, take state.params as POST body and save the response to state.data after request finished
  3. addRow Add state.data to the data source of current table view

Now the pipeline cannot work yet, we needs to mock a POST request

POST heroes

and return a mock response.

Run the following command to create a mock JSON:

pbind mock PBClient heroes@post

Modify the generated heroes@post.json:

{
  "$redirect": "#.params"
}

Note the keyword $redirect here by which we will redirect to the data as response. And the #.paramsm here will map to the action state parameters that means the submitting parameters of the form.
