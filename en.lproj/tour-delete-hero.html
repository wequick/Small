Delete hero

First, let's reduce the problem:

  1. Use the swipe-to-delete interaction of the UITableView
  2. Send a DELETE request
  3. When we got a 204 (No Content) status, remove the row of UITableView

We can also use the Action to pipe the stream.

Swipe to delete

Add a delete action for the Root > properties > row > actions node in the main.tableview.plist:

KeyTypeValue
deleteDictionary(3 items)
typeStringdelete
nameStringheroes/:id
next.doneDictionary(1 item)
typeStringdeleteRow

Note the first delete here refer to the Row Action by configure which the swipe interaction of the cell would be actived. And the second delete is for sending the DELETE request, the last deleteRow used to remove the row on UI.

Mock request

The API:

DELETE heroes/:id

The mocking:

pbind mock PBClient heroes/:id@delete

Modify the generated heroes-:id@delete.json:

{
  "$status": 204
}

Note the keyword $status here will mock the response status code.

Remove row

When the status code turns to be 204, the deleteRow action is actived and remoe the editing cell of the table view automatically.
