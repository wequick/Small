Required

By last section we had accomplished a form, but as you see when the name of the hero is empty we also can submit the form. We needs to add some limits for this.

Back to the hero-new.form.plist, let's add a required property for the name input:

KeyTypeValue
@input.requiredBooleanYES

Now it works and we gonna to show an error tips for it to make the application more friendly.

Before this let's make a little tweaking with the configuration cause we just got too many @input.** properties, we needs to extract the @input:

KeyTypeValue
propertiesDictionary(2 items)
@title.textStringName
@inputDictionary(3 items)
nameStringname
placeholderStringName
requiredBooleanYES

well, much more clear.

Error tips

Now let's configure an errorRow for the @input:

KeyTypeValue
errorRowDictionary(1 item)
layoutStringinput.error

and then create the error layout input.error.plist:

KeyTypeValue
RootDictionary(3 items)
viewsDictionary(2 items)
iconDictionary(2 items)
clazzStringUILabel
propertiesDictionary(2 items)
textString*
textColorString#ff0000
tipsDictionary(2 items)
clazzStringUILabel
propertiesDictionary(1 item)
textColorString#ff0000
metricsDictionary(1 item)
leftString116
formatsArray(3 items)
Item 0String|-left-[icon][tips]
Item 1StringV:|[icon]|
Item 2StringV:|[tips]|

Note the view alias tips here would be recognized by errorRow as the default label to display error tips.

Save and try to submit a empty name, then we would got an errow row. It display "Required" here cause that's the default error tips built-in Pbind, you can customize it by requiredTips:

KeyTypeValue
requiredTipsStringName is required

Save the file:

What's more, you may wanna hide the error after you input something correctly. This requires the opening of instant validating feature of form.

Add validating property in the Root > properties node of hero-new.form.plist:

KeyTypeValue
validatingString:changed

Save:

We can configure other limits like max-length, regex and etc. for more details please check the Input component.
